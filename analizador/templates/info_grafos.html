{% translate "TEORIA DE GRAFOS" %}

{% translate "Definición" %}

{% blocktranslate %} Un grafo es una estructura matemática que representa relaciones entre un conjunto de objetos. Se define formalmente como un par ordenado: {% endblocktranslate %}

G=(V,E)

{% translate "Donde:" %}

  • V {% translate "es un conjunto finito de vértices o nodos." %}
  • {% blocktranslate %} E es un conjunto de aristas o edges, que conectan pares de vértices. {% endblocktranslate %}

{% blocktranslate %} Vecindad: Dos nodos son adyacentes si existe una arista que los une. {% endblocktranslate %}

{% blocktranslate %} Distancia: La distancia entre dos nodos u y v es la mínima cantidad de ejes por los que debo moverme para llegar de u a v. {% endblocktranslate %}

{% translate "Historia de la Teoría de Grafos" %}

{% blocktranslate %} El origen de la teoría de grafos se remonta al siglo XVIII con el problema de los puentes de Königsberg, planteado por Leonhard Euler en 1736. Euler demostró que no existía un camino que recorriera todos los puentes sin repetirlos, sentando así las bases de la teoría de grafos y la topología. {% endblocktranslate %}

{% blocktranslate %} En 1847, Gustav Kirchhoff aplicó la teoría de grafos en circuitos eléctricos, formulando las leyes de Kirchhoff. {% endblocktranslate %}

{% blocktranslate %} En 1852, Francis Guthrie propuso el problema de los cuatro colores, resuelto en 1976. En 1857, Arthur Cayley utilizó los grafos para contar isómeros químicos. {% endblocktranslate %}

{% blocktranslate %} El primer libro sobre teoría de grafos fue escrito por Dénes Kőnig en 1936. {% endblocktranslate %}

{% blocktranslate %} Desde la década de 1950, los grafos se han usado ampliamente en sociometría y redes sociales, permitiendo analizar relaciones entre personas y estructuras organizacionales. {% endblocktranslate %}

{% translate "Grafo Simple" %}

{% translate "Permite múltiples aristas entre dos vértices." %}

{% blocktranslate %} Las aristas tienen una dirección específica representada con flechas. {% endblocktranslate %}

{% translate "Las aristas tienen un peso o valor numérico." %}

{% translate "Existe un camino entre cualquier par de vértices." %}

{% translate "REPRESENTACIÓN DE GRAFOS" %}

{% blocktranslate %} Los grafos pueden representarse de diversas maneras en la informática y la teoría de grafos. Las dos formas más comunes son matriz de adyacencia y lista de adyacencia. Cada una tiene sus ventajas y desventajas dependiendo del tipo de grafo y del problema que se quiera resolver. {% endblocktranslate %}

{% translate "Matriz de Adyacencia" %}

{% blocktranslate %} Es una matriz cuadrada de tamaño N×N (donde N es el número de vértices) en la que cada celda indica si hay una arista entre dos vértices. {% endblocktranslate %}

  • {% blocktranslate %} Si en la posición (i,j) hay un 1, significa que existe una conexión entre el nodo i y el nodo j. {% endblocktranslate %}
  • {% translate "Si hay un 0, significa que no existe tal conexión." %}

{% translate "Lista de Adyacencias" %}

{% blocktranslate %} La lista de adyacencia es una representación de un grafo en la que cada nodo tiene asociado un conjunto (o lista) de nodos adyacentes, es decir, aquellos con los que tiene una conexión directa mediante una arista. {% endblocktranslate %}

{% translate "En esta estructura:" %}

  • {% blocktranslate %} Se utiliza un array o diccionario, donde cada índice o clave representa un nodo. {% endblocktranslate %}
  • {% blocktranslate %} Cada entrada contiene una lista con los nodos a los que está conectado. {% endblocktranslate %}

{% translate "ALGORITMO DE"%} DIJKSTRA

{% blocktranslate %} El algoritmo de Dijkstra es un método eficiente para encontrar el camino más corto desde un nodo de origen a todos los demás nodos en un grafo ponderado, siempre que los pesos de las aristas sean no negativos. Fue desarrollado en 1956 por el científico informático Edsger W. Dijkstra y se ha convertido en una herramienta fundamental en teoría de grafos y optimización de rutas. {% endblocktranslate %}

{% translate "Pasos del Algoritmo" %}
  • {% translate "Inicialización:" %}

    {% blocktranslate %} Se establece la distancia del nodo origen en 0 y la de los demás nodos en infinito. Se usa una cola de prioridad para procesar los nodos con menor distancia conocida. {% endblocktranslate %}

  • {% translate "Exploración de vecinos:" $}

    {% blocktranslate %} Se extrae el nodo con la menor distancia acumulada de la cola. Se recorren sus vecinos y, si se encuentra una ruta más corta hacia ellos, se actualiza su distancia. Se inserta nuevamente en la cola de prioridad si su distancia cambia. {% endblocktranslate %}

  • {% translate "Finalización:" %}

    {% blocktranslate %} El algoritmo se detiene cuando todos los nodos han sido procesados o cuando se alcanza un nodo destino específico (si se busca solo un camino mínimo en particular). Se puede reconstruir el camino más corto almacenando los predecesores de cada nodo. {% endblocktranslate %}

{% translate "Complejidad Computacional" %}

{% blocktranslate %} La complejidad del algoritmo depende de la estructura de datos utilizada para manejar la cola de prioridad: {% endblocktranslate %}

  • {% translate "Usando una matriz de adyacencia:" %} O(V²)
  • {% translate "Usando un heap binario:" %} O((V+E) log V)
  • {% translate "Usando un Fibonacci Heap: " %} O(E + V log V)
